# Политика конфиденциальности мессенджера «Лично»

*Действует с 12 июля 2026 г.*

«Лично» — мессенджер со сквозным (end-to-end) шифрованием. Приложение спроектировано так, чтобы **не собирать персональные данные**: в нём нет учётных записей, регистрации, номеров телефонов, адресов электронной почты, аналитики и рекламы.

## 1. Какие данные мы собираем

**Никакие.** У приложения нет серверной базы пользователей, и разработчик не получает, не хранит и не передаёт третьим лицам никаких сведений о вас. Мы технически не можем прочитать вашу переписку, посмотреть список ваших контактов или узнать, кто вы.

## 2. Что хранится только на вашем устройстве

- **Ключи шифрования.** Создаются на устройстве при первом запуске и никогда его не покидают. Секретные ключи лежат в защищённом хранилище системы (Android Keystore).
- **Переписка** (сообщения, фото, файлы, голосовые) — хранится локально в зашифрованном профиле приложения.
- **Контакты** — только те, с кем вы обменялись QR-кодами лично. Приложение **не запрашивает доступ** к адресной книге телефона.
- **Имя и фото профиля** — задаются вами по желанию и передаются в зашифрованном виде только вашим контактам.

Удаление приложения удаляет все эти данные. Резервная копия ключа создаётся только вручную и зашифрована фразой, которую знаете только вы.

## 3. Что видят серверы-ретрансляторы (релеи)

Сообщения передаются через сеть независимых серверов-ретрансляторов. Из-за принципа работы сети релей технически обрабатывает:

- **зашифрованные конверты сообщений** — их содержимое (текст, вложения, имена файлов, факт наличия вложения) серверу недоступно: расшифровать их может только устройство получателя;
- **публичные ключи** отправителя и получателя — случайные идентификаторы, не связанные с личностью, номером телефона или другими данными;
- **IP-адрес подключения** — как любой сервер в интернете, транзитно, для поддержания соединения;
- **push-токен** (если разрешены уведомления) — чтобы разбудить приложение при входящем сообщении. Само push-уведомление **не содержит** ни текста, ни имени отправителя.

Недоставленные зашифрованные конверты хранятся на релее до получения адресатом, но не дольше 14 дней, после чего удаляются автоматически.

## 4. Разрешения приложения

- **Камера** — только для сканирования QR-кода контакта и видеозвонков/кружков;
- **Микрофон** — только для голосовых сообщений и звонков;
- **Фото/файлы** — только когда вы сами прикрепляете вложение;
- **Уведомления** — для оповещения о входящих сообщениях.

Ни одно разрешение не используется в фоне для сбора данных.

## 5. Третьи стороны

Для доставки push-уведомлений используется Firebase Cloud Messaging (Google). Через него проходит только техническое событие «есть новое сообщение» без содержимого и без сведений об отправителе. Других сторонних сервисов, SDK аналитики и рекламных сетей в приложении нет.

## 6. Дети

Приложение не предназначено для детей младше 12 лет и не собирает данные ни о каких пользователях, включая детей.
